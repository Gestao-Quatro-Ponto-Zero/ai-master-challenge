# 🧠 AI Development Process Log

This folder documents the structured development of the Churn Intelligence System using AI-assisted engineering.

## 📊 1. Business Understanding

- Churn defined as financial risk (not only customer count)
- Revenue segmentation using Pareto (ABC)
- Identification of high-impact accounts

---

## 🏗️ 2. Solution Design

- Modular architecture (API + ML + AI + Data Pipeline)
- Executive KPI modeling
- Separation between dashboard and churn risk module

---

## ⚙️ 3. Backend Engineering

- Data ingestion via Kaggle dataset
- Feature engineering pipeline
- FastAPI endpoints for KPIs, churn risk and AI insights

## 🤖 4. Machine Learning

- XGBoost model for churn prediction
- Feature set based on usage, revenue and support signals
- Output: churn_score (probability)

## 🔍 5. Explainability (SHAP)

- Identification of key churn drivers:
  - Usage drop
  - Support issues
  - Engagement patterns

  - Churn distribution
