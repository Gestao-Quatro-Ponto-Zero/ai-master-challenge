====================

O Score Size será **apenas estrutural**.

NORMALIZAÇÃO E LÓGICA DE CÁLCULO
================================

1\. Base histórica para cálculo
-------------------------------

Os cálculos de taxa de ganho e tempo médio devem ser feitos com base nos leads **fechados**, ou seja:

*   `deal_stage = Won`
*   `deal_stage = Lost`

Não usar Prospecting e Engaging para calcular win rate ou tempo médio histórico.

2\. Taxa de ganho histórica
---------------------------

Para cada grupo analisado, calcular:

`win_rate = won / (won + lost)`

3\. Tempo médio histórico
-------------------------

Para cada grupo analisado, calcular:

`tempo_fechamento = close_date - engage_date`

Usar em dias.

4\. Buckets de porte da empresa
-------------------------------

Classificar `employees` em 4 faixas:

*   0 a 250
*   251 a 1000
*   1001 a 5000
*   acima de 5000

Essas faixas devem ser usadas para calcular as métricas históricas do porte.

5\. Regra de normalização
-------------------------

Como win rate alto é melhor e tempo baixo é melhor, ambas as métricas devem ser convertidas para a mesma lógica: **quanto maior a nota, melhor**.

Sugestão obrigatória de abordagem:

*   transformar cada uma das 4 variáveis em um score padronizado de 0 a 100
*   para win rate: quanto maior o win rate, maior a nota
*   para tempo de fechamento: quanto menor o tempo, maior a nota

6\. Fórmula final do score estrutural
-------------------------------------

Calcular:

`score_size_numerico = media( nota_segmento_winrate + nota_segmento_tempo + nota_porte_winrate + nota_porte_tempo )`

Ou seja:

`score_size_numerico = (v1 + v2 + v3 + v4) / 4`
