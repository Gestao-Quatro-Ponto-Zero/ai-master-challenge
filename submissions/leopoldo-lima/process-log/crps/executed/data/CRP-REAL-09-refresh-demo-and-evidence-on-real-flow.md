# CRP-REAL-09 — Refazer demo e evidências usando o fluxo real

## Objetivo
Alinhar a demonstração do produto ao fluxo real com dataset oficial.

## Problema que resolve
Mesmo com produto corrigido, demo velha ou evidência desatualizada pode trair a submissão.

## Tarefas
1. Atualizar roteiro de demo.
2. Capturar novas screenshots e evidências do fluxo real.
3. Atualizar artifacts e evidências de PR.
4. Garantir que qualquer gravação, print ou script reflita o runtime real.

## Entregáveis
- `docs/DEMO_SCRIPT.md` atualizado
- screenshots atualizadas em `artifacts/`
- atualização de `PROCESS_LOG.md`
- atualização de `LOG.md`

## Impacto na Submissão
- fortalece a percepção de produto pronto para uso amanhã
- evita inconsistência entre código e apresentação

## Evidências obrigatórias
- screenshots da UI real
- exemplo de chamada de API real
- roteiro de demo revisto
- artefatos salvos no repositório

## Atualizações obrigatórias de process log
- registrar o que mudou na demo
- explicar por que evidências antigas foram substituídas
- anotar revisão humana do material final

## Atualizações obrigatórias de README/Submission
- atualizar links para demo, screenshots e comandos

## Definition of Done
- demo e evidências mostram o produto real
- não restam capturas enganosas do modo demo
- LOG e PROCESS_LOG atualizados
