# CRP-DOC-03 — Índice consolidado de evidências

## Objetivo
Criar um único índice navegável para todas as evidências que sustentam a submissão.

## Tarefas
1. Criar `docs/EVIDENCE_INDEX.md`.
2. Indexar:
   - screenshots
   - vídeo
   - chat exports
   - notas de decisão
   - test runs
   - IA trace
   - git history / commits relevantes
3. Identificar arquivos inexistentes com status `faltante`.
4. Atualizar README da submissão com link para o índice.
5. Atualizar `LOG.md` e `PROCESS_LOG.md`.

## Entregáveis
- `docs/EVIDENCE_INDEX.md`
- README atualizado
- logs atualizados

## Impacto na Submissão
Facilita a auditoria do process log e evita que o avaliador tenha de caçar provas.

## Evidências obrigatórias
- índice publicado
- cada item com path ou link
- status presente/faltante documentado

## Definition of Done
- índice publicado
- vídeo e screenshots aparecem no índice
- process log e IA trace aparecem no índice
- logs atualizados
