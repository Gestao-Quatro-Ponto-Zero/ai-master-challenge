# CRP-UX-06 — Resgatar seletivamente o que prestava do Lovable

## Objetivo
Aproveitar o melhor da trilha Lovable sem reintroduzir o domínio errado.

## Problema
O runtime atual perdeu força visual e experiencial. Já existe material anterior com badges, tokens e cards melhores, mas ele não pode voltar inteiro porque também carregava semântica incorreta.

## Tarefas
1. Inspecionar a trilha Lovable legada.
2. Catalogar ativos reaproveitáveis:
   - badges de prioridade
   - badges de estágio
   - cards de explicação
   - padrões de densidade visual
3. Reaproveitar apenas o que:
   - melhora a UX
   - não reintroduz `Proposal`, `Negotiation` etc.
4. Registrar claramente o que foi resgatado e o que foi descartado.

## Impacto na Submissão
- Recupera força de produto sem aumentar ruído técnico.
- Aumenta percepção de acabamento.

## Evidências obrigatórias
- lista dos ativos reaproveitados
- screenshots comparativas
- diff dos componentes visuais resgatados
- nota humana justificando cada reaproveitamento

## Atualizações obrigatórias de process log
Registrar:
- o que existia no Lovable
- o que foi descartado por semântica errada
- o que foi incorporado
- por que isso melhora a submissão

## Atualizações obrigatórias de README/Submission
- atualizar screenshots finais se houver melhoria visual relevante

## Definition of Done
- UI visualmente mais forte
- nenhuma regressão de domínio
- reaproveitamento seletivo documentado
- `PROCESS_LOG.md` atualizado
- evidências salvas
- README atualizado quando aplicável
- verificação humana explícita realizada
