# CRP-UX-03 — Reestruturar a tela para cockpit de decisão

## Objetivo
Parar de parecer página técnica com três blocos genéricos e virar cockpit comercial.

## Problema
A tela atual tem:
- muito espaço vazio
- pouca densidade útil
- foco visual fraco
- hierarquia de decisão ruim

## Tarefas
1. Reorganizar o layout com foco em:
   - KPIs no topo
   - ranking como elemento central
   - detalhe em painel ou área nobre
   - filtros mais compactos
2. Melhorar densidade informacional sem poluir.
3. Reforçar hierarquia visual:
   - score
   - prioridade
   - próxima ação
4. Eliminar sensação de “shell técnica”.

## Impacto na Submissão
- Aumenta percepção de usabilidade e prontidão.
- Ajuda a tornar a solução acionável amanhã.

## Evidências obrigatórias
- screenshot antes/depois
- comparação comentada
- nota humana com decisão de layout
- se possível, gravação curta de navegação

## Atualizações obrigatórias de process log
Registrar:
- problemas da hierarquia anterior
- racional do novo layout
- referências reaproveitadas
- o que foi descartado

## Atualizações obrigatórias de README/Submission
- atualizar imagens da solução
- ajustar a descrição da interface final

## Definition of Done
- layout com hierarquia clara
- ranking central e legível
- detalhe não secundário
- filtros menos burocráticos
- `PROCESS_LOG.md` atualizado
- evidências salvas
- README atualizado quando aplicável
- verificação humana explícita realizada
